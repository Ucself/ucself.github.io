前言

经常使用别人的封装库或者项目，在项目开发中自己也有封装，即使不能给别人广泛使用，也可以自己在多项目中 pod 方式使用自己封装的代码 前提：mac 上已经装好了 pod,git

一、创建自己库添加 Cocoapods 支持

  • 基于 pod 自动创建项目

在终端输入

pod lib create JGCache

提示对应的询问命令

Cloning `https://github.com/CocoaPods/pod-template.git` into `JGCache`.
Configuring JGCache template.

------------------------------

To get you started we need to ask a few questions, this should only take a minute.

If this is your first time we recommend running through with the guide:
 - http://guides.cocoapods.org/making/using-pod-lib-create.html
 ( hold cmd and double click links to open in a browser. )


What language do you want to use?? [ Swift / ObjC ]
 > Swift

Would you like to include a demo application with your library? [ Yes / No ]
 > Yes

Which testing frameworks will you use? [ Quick / None ]
 > None

Would you like to do view based testing? [ Yes / No ]
 > No

项目创建成功，并编译成功

  • 接下来在https://github.com/官网自己的账号下创建一个仓库JGCache（这里不是重点，不用介绍了）

  • 将 pod 命令自动创建的项目上传到 github JGCache 项目中，（这里不是重点，不用介绍了）

  • 给仓库 JGCache 主分支 master 创建一个 tag 0.1.0 （这里不是重点，不用介绍了）

  • 检测填写的 podspec 是否有错误，cd 进入 JGCache 文件目录
pod spec lint
  • 测试本地导入的文件是否报错。cd 进入项目 Example 文件路径,导入依赖的第三方库
pod install
  • 上传的 cocopods 管理
pod trunk push JGCache.podspec --allow-warnings

二、更新自己库添加 Cocoapods 支持

  • JGCache.podspec 文件中你需要更改的项目更改（主要更改版本如：1.0.0）

  • 在啊 github 对你的工程添加标签如 1.0.0

  • 上传工程工程代码

  • 再次执行

pod trunk push JGCache.podspec --allow-warnings
